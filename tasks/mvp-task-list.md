# MVP 开发任务清单

## 0. MVP 边界

第一版只跑通：

```text
选择题 → 3 个方向 → 3 个选题 → 1 条内容文案 → 反馈改写 → 小光沉淀故事 → 故事库展示
```

第一版不做：账号完整规划、30 条选题、内容自动发布、短视频剪辑、复杂多模型路由、复杂故事库编辑器、涨粉/变现承诺。

---

## 1. 首页

- [ ] 展示「嘿，你来了」
- [ ] 主按钮：生成一条内容文案
- [ ] 次入口：我的故事库
- [ ] 每页右下角固定小光漂浮入口

---

## 2. 三道选择题

### 第 1 题：素材来源
- [ ] 问题：你最近最常投入时间的事情是什么？
- [ ] 选项：工作/职业、读书/学习、带孩子/家庭、生活记录、某个爱好、副业/赚钱、情绪/成长、我也说不清

### 第 2 题：内容目标
- [ ] 问题：你更想通过内容获得什么？
- [ ] 选项：打造个人 IP、赚钱变现、获得客户、记录成长、分享经验、表达自己

### 第 3 题：发布载体
- [ ] 问题：你想先把这条内容发在哪里？
- [ ] 选项：短视频/口播、小红书/图文、公众号/文章、头条号/文章、朋友圈/动态、我还不确定

---

## 3. 方向推荐

- [ ] 调用 `recommend_direction` Prompt
- [ ] 生成 3 个 IP 内容方向
- [ ] 其中 1 个标记「更适合你」
- [ ] 每个方向包含：方向名称、这个方向讲什么、适合吸引谁、为什么适合你
- [ ] 支持「都不太像我」「换一批方向」「我想补充一句」

---

## 4. 选题推荐

- [ ] 调用 `recommend_topic` Prompt
- [ ] 基于用户选定方向生成 3 个第一条内容选题
- [ ] 其中 1 个标记「更适合你」
- [ ] 支持「换一批选题」「我想自己说个主题」

---

## 5. 内容文案生成

- [ ] 调用 `generate_content` Prompt
- [ ] 根据发布载体输出不同结构
- [ ] 短视频：标题、开头 3 秒、正文口播、结尾引导、拍摄提示
- [ ] 小红书：标题、封面文案、正文、结尾互动、话题标签
- [ ] 公众号/头条：标题、开头、正文结构、正文内容、结尾观点/互动
- [ ] 朋友圈：正文文案、配图建议、轻互动结尾

---

## 6. 反馈改写

- [ ] 文案下方反馈按钮：不够像我、太官方了、太营销了、太长了、太短了、开头不吸引人、换一种讲法
- [ ] 点击后调用 `revise_content` Prompt
- [ ] 不让用户重新走完整流程

---

## 7. 小光

- [ ] 每页右下角漂浮入口
- [ ] 点开后第一句话：想起什么，都可以跟我说。
- [ ] 支持用户自由讲故事
- [ ] 生成沉淀卡：经历、想法、金句、可生成选题
- [ ] 第一次使用时展示「开启自动沉淀我的故事」开关
- [ ] 注意隐私表达：只保存用户愿意留下来的内容

---

## 8. 故事库

- [ ] 首页进入「我的故事库」
- [ ] 四个分类：经历、想法、金句、选题
- [ ] 展示成长数量统计
- [ ] 第一版只做查看、复制、基于卡片生成文案
- [ ] 第一版不做编辑、删除、合并、复杂搜索

---

## 9. 模型服务层

- [ ] 新建 `modelService.generate(taskType, payload)`
- [ ] 第一版默认只接 DeepSeek
- [ ] 代码结构预留未来接 OpenAI / Claude / Gemini
- [ ] 不做复杂多模型路由
