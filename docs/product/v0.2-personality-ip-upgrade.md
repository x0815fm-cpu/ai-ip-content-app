# V0.2 产品升级：从「生成文案」到「建立人格 IP」

日期：2026-07-10  
状态：产品设计稿 / Codex 后续执行依据  
适用阶段：PR #6 合并后的下一阶段

---

## 1. 本轮升级目的

当前 MVP 已经能够完成：

```text
首页
↓
三道选择题
↓
3 个内容方向
↓
3 个选题
↓
生成一条内容文案
↓
反馈改写
↓
小光沉淀故事
```

V0.2 不推翻现有 MVP，而是在现有流程上明确长期方向：

> 产品不只是帮助用户生成内容，而是帮助用户通过持续表达，把自己的人格魅力被同频的人看见，最终建立长期个人 IP。

短视频、文案、选题、故事库、小光聊天，都是为这个目标服务的手段。

---

## 2. 第一批目标用户

第一批用户不是泛普通人，而是：

> 拆书会里想做个人 IP 的会员。

他们通常具备：

- 有持续学习与读书习惯。
- 有真实人生经历和思考。
- 想通过短视频表达自己。
- 不清楚自己该讲什么。
- 不知道第一条视频怎么开始。
- 希望通过内容吸引同频的人。
- 最终可以通过带来拆书会会员获得佣金。

因此产品的第一阶段商业闭环是：

```text
真实表达
↓
吸引同频的人
↓
带入拆书会
↓
获得佣金
↓
建立表达信心
↓
继续输出
```

---

## 3. 产品最高目标

如果用户使用产品 6 个月，成功不只体现在粉丝数。

产品需要同时支持两种结果：

### 3.1 外在结果

- 用户通过短视频表达吸引同频的人。
- 用户能够带来拆书会新会员。
- 用户获得阶段性佣金或其他真实回报。

### 3.2 内在结果

- 更敢表达。
- 更清楚自己是谁。
- 更知道自己能讲什么。
- 更相信自己的经历有价值。
- 逐渐形成稳定的人格 IP。

产品原则：

> 状态改变和变现是相辅相成的。状态让表达更真实，阶段性变现让用户获得信心。

---

## 4. 长期 IP 的核心

长期 IP 最重要的不是单一专业标签，而是：

> 人格魅力。

人格魅力是以下内容的总和：

- 真实经历
- 认知变化
- 价值观
- 专业能力
- 面对困境的方式
- 面对失败后的选择
- 持续成长的过程

因此，产品不应只帮助用户展示「我有多厉害」，而要帮助用户表达：

> 我曾经怎么困住、怎么失败、怎么转向、为什么成为今天的自己。

---

## 5. 第一条视频的内容原则

第一条视频优先从：

> 一次真实的内心失败

开始。

优先级高于：

- 成功经历
- 专业成就
- 泛知识分享
- 热点观点
- 营销技巧

### 5.1 为什么选择真实失败

一次真实失败更容易展示：

- 脆弱
- 选择
- 信念
- 变化
- 人格

失败不是为了卖惨，而是为了寻找：

> 用户在失败前相信什么，失败时经历了什么，失败后改变了什么。

### 5.2 优先寻找的失败类型

第一版优先引导：

> 自己内心的失败。

例如：

- 我第一次承认自己很焦虑。
- 我发现自己一直活在别人的期待里。
- 我觉得自己可能真的做不到。
- 我曾经想放弃。
- 我在很长时间里疲惫、麻木、没有方向。

工作、家庭等外部事件可以作为背景，但核心要落到用户内心发生了什么。

---

## 6. 转折点原则

产品需要逐步建立一个核心能力：

> 转折点发现器（Turning Point Finder）

小光面对用户故事时，不只总结事件，而要寻找：

```text
低谷
+
选择
+
变化
+
新的信念
=
转折点
```

小光需要尝试理解：

- 用户原来相信什么？
- 哪件事击碎了原来的相信？
- 用户做了什么选择？
- 用户后来形成了什么新的认知？
- 这个变化如何影响今天的用户？

产品表达原则：

> 人格魅力不是从「我很成功」里出来的，而是从「我如何走过低谷」里出来的。

---

## 7. 第一条视频的建议结构

未来正式 Prompt 应优先使用以下内容结构：

```text
1. 开头：一个真实、具体的失败瞬间
2. 困境：当时内心最真实的感受
3. 转折：发生了什么，让用户开始改变
4. 认知：失败前后，用户相信的东西发生了什么变化
5. 当下：这段经历如何影响今天的用户
6. 价值：这段故事能给同频的人什么启发
7. 钩子：自然提到读书、拆书会、同伴或组织支持
```

钩子不是硬广告。

理想状态是：

> 先用真实案例建立信任，再轻轻告诉观众，自己是在怎样的学习和组织中慢慢变好的。

---

## 8. 内容与转化的关系

单条视频可以同时包含：

```text
真实案例
↓
人格表达
↓
给别人启发
↓
轻量招募
```

长期账号可以分阶段：

### 8.1 前期：立住人格

内容重点：

- 真实失败
- 内心困境
- 读书后的变化
- 对生活的理解
- 自己为何开始成长

### 8.2 后期：自然转化

内容重点：

- 拆书会的日常
- 晨读带来的改变
- 同伴与老师的支持
- 社群里的真实案例
- 邀请同频的人一起加入

第一版仍然只生成一条视频文案。

一个月规划属于后续能力，必须建立在用户对第一条结果产生信任之后。

## 10. 小光的短期与长期关系

### 10.1 短期关系

用户回来是因为：

> 我又来了，再帮我出一个新文案。

此时小光提供的是工具价值。

### 10.2 长期关系

用户回来是因为：

> 我又来了，碰到了一件事，我想听听你的看法。

此时小光提供的是关系价值。

长期目标不是把小光做成泛聊天机器人，而是让小光越来越理解：

- 用户经历过什么
- 用户相信什么
- 用户为何改变
- 用户适合表达什么
- 用户的人格魅力在哪里

## 11. 故事库 V2 方向

当前故事库包含：

- 经历
- 想法
- 金句
- 选题

未来建议升级为「人格资产库」：

```text
人物资产
├── 转折点
├── 长期困境
├── 信念变化
├── 金句
├── 故事
├── 可表达主题
└── 内容资产
```

### 11.1 转折点建议字段

```ts
type TurningPointAsset = {
  id: string;
  title: string;
  failureMoment: string;
  innerStruggle: string;
  previousBelief: string;
  triggerEvent: string;
  choiceMade: string;
  newBelief: string;
  currentImpact: string;
  possibleTopics: string[];
  createdAt: string;
};
```

注意：本轮先作为产品设计，不要求一次性开发全部字段。

## 12. Prompt 后续升级方向

### 12.1 recommend_direction

不要只推荐：

- 个人成长
- 职场经验
- 家庭教育
- AI 学习

这些过于宽泛。

方向应更贴近用户真实经历和人格表达，例如：

- 一个长期自我怀疑的人，如何重新建立行动感
- 从一次想放弃的经历里，讲清自己为什么开始读书
- 把自己在迷茫中的变化讲给同样困住的人

方向必须：

- 具体
- 真实
- 可马上表达
- 能长期延展
- 有人格感

### 12.2 generate_content

以后生成内容时，应尽量优先寻找：

- 一次真实失败
- 内心困境
- 转折点
- 信念变化
- 当下意义
- 自然钩子

### 12.3 extract_story_asset

未来不只提取经历、想法、金句、选题，还要尝试提取：

- 失败瞬间
- 内心困境
- 原有信念
- 触发事件
- 新信念
- 转折点

## 13. 建议开发拆分

不要一次性开发全部 V0.2。

建议拆成三个独立 PR。

### PR-07：Turning Point Input V1

目标：在生成第一条文案前，增加一次真实失败的轻量输入。

建议范围：

- 在选题之后、生成文案之前增加一张输入卡。
- 引导问题：
  - 「有没有一次你真的觉得自己不行了？」
  - 「当时你心里最真实的感受是什么？」
- 用户可以输入一小段文字。
- 输入内容传给 `generate_content`。
- 保留跳过选项，避免阻断现有流程。

本 PR 不做复杂对话，不做数据库。

### PR-08：Generate Content Prompt V2

目标：围绕真实失败和转折点生成第一条短视频文案。

建议范围：

- 新增 `prompts/generate_content.md`。
- Prompt Registry 接入 `generate_content`。
- 更新 payload：增加 `turningPointInput`。
- 输出继续符合现有 `GeneratedContent` 结构。
- 文案包含真实案例、转折、认知和轻量钩子。

### PR-09：Story Asset V2

目标：让小光从用户输入中提取转折点资产。

建议范围：

- 扩展 `ExtractedStoryAsset`。
- 增加 turningPoint 字段。
- 更新 `extract_story_asset` Prompt。
- 故事库先展示一张「转折点」卡。
- 暂不做持久化数据库。

---

## 14. PR-07 的建议验收标准

下一轮建议先做 PR-07。

验收标准：

1. 原有流程仍然可运行。
2. 用户选定选题后，出现一次真实失败的轻量输入页。
3. 输入页不强迫用户写长故事。
4. 用户可以跳过。
5. 输入内容能进入 `generate_content` payload。
6. 当前 mock 流程不报错。
7. 不改故事库，不改 DeepSeek provider，不写其他 Prompt。
8. `typecheck / lint / build` 全部通过。

## 16. 北极星原则

以后所有功能都要回答：

> 这个功能是否帮助用户建立长期个人 IP？

产品最终不是帮助用户大量生产内容，而是：

> 帮助用户发现真实的自己，持续表达真实的自己，把人格魅力被同频的人看见，最终建立一个有生命力的长期个人 IP。
