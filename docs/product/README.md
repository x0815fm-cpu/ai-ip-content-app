# Product 文档区

这里存放 APP 的产品事实源与产品架构说明。

## 目录职责

本目录主要回答：

> 这个产品是什么？为什么这么设计？后续要往哪里走？

适合存放：

- PRD / 产品说明
- 产品架构
- Router 设计
- 页面流程
- MVP / Beta / V1 路线图
- 数据结构说明
- Prompt 与产品逻辑之间的关系

## 管理原则

1. **GitHub 仓库是唯一事实源。**
2. 涉及产品方向、流程、状态机、Router、Prompt、故事库定位的修改，必须先更新对应文档，再改代码。
3. 不把临时想法直接写进代码；先沉淀成文档，再交给 Codex 实现。
4. MVP 阶段保持轻，不做复杂 Multi-Agent，不做过度架构。

## 当前核心产品定义

> AI 用选择题主动引导普通人找到适合自己的 IP 内容方向，并生成第一条可发布内容文案。

## 当前 MVP 主流程

```text
首页
↓
三道选择题
↓
3 个内容方向
↓
3 个选题
↓
生成一条内容文案
↓
反馈改写
↓
小光沉淀故事
↓
故事库展示
```
