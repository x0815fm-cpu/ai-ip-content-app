# Review 009：PR-08 空故事场景第二轮真实模型验收

日期：2026-07-10  
评审对象：PR #8 `feat: implement recommend_topic prompt v2 with validation`  
依据：
- `docs/reviews/review-006-pr-08-technical-plan.md`
- `docs/reviews/review-008-pr-08-real-model-acceptance.md`

结论：**仍未通过，暂不合并。**

## 1. 本轮已经完成的修复

Prompt 已增加空故事场景的真实性硬约束：

- 不得断言未提供的具体事件已经发生；
- 不得补充时间、人物、场景、失败、评价、结果或变化；
- 选题必须能从 payload 中直接找到依据；
- 允许开放式、反思式、动机式标题；
- “具体、第一人称”不等于虚构具体事件。

设计稿与运行时 Prompt 的新增规则保持一致。

## 2. 第二轮真实输出复核

已知输入只有：

- 素材来源：工作/职业；
- 内容目标：记录成长；
- 发布渠道：小红书/图文；
- 已选方向：职场成长实录；
- `failureStoryInput: ""`。

模型返回：

1. `我为什么开始记录工作日常`
2. `工作里让我想改变的一个瞬间`
3. `职场第一年，我学会的一件小事`

### 2.1 第一条

`我为什么开始记录工作日常`

基本可接受。它与“记录成长”和“职场成长实录”直接相关，虽然“已经开始”比“想开始”更强，但没有引入具体时间、人物或失败事件。

### 2.2 第二条

`工作里让我想改变的一个瞬间`

未通过。它断言用户存在一个已经发生的、足以触发改变的具体瞬间，而 payload 没有提供这一事实。

安全写法应为：

- `工作里，我现在最想改变什么`
- `我想改变怎样的工作状态`

### 2.3 第三条

`职场第一年，我学会的一件小事`

未通过。它同时虚构了：

- 用户处于或经历过“职场第一年”；
- 用户已经“学会”某件事；
- 存在一个可以讲述的具体小事。

这些都无法从 payload 中得到依据。

安全写法应为：

- `我想记录怎样的职场成长`
- `关于工作，我现在最想说的一件事`

## 3. 根因

Prompt 中仍存在规则冲突。

空故事场景要求：

> 不得虚构具体事件。

但通用选题要求仍写着：

> 不要太宏大，聚焦一个具体场景或感受。

模型为了满足“具体场景”，继续补造“一个瞬间”“职场第一年”等具体经历。

因此本轮不能继续只增加禁止条款，必须把通用规则改成条件化规则。

## 4. 必须修订的 Prompt

只修改：

- `prompts/recommend_topic.md`
- `src/services/prompts/promptRegistry.ts`

两处保持一致。

### 4.1 将通用规则拆成条件规则

删除或改写：

```text
不要太宏大，聚焦一个具体场景或感受。
```

改为：

```text
- 有 failureStoryInput 时：优先聚焦 payload 中已经出现的具体场景、时刻或感受。
- 无 failureStoryInput 时：不要创造具体场景；应聚焦 payload 已明确提供的方向、动机、目标、当前状态或想表达的问题。
```

### 4.2 空故事时禁止过去事件断言

增加：

```text
当 failureStoryInput 为空时，标题不得使用会暗示既有经历的表达，除非 payload 明确提供，例如：
“一次”“那天”“那个瞬间”“第一周”“第一年”“后来”“终于”“被否定”“被批评”“搞砸”“学会了”“差点”。
如果拿不准事实是否存在，必须改写成“我想……”“我现在最想……”“我为什么想……”“对我来说……意味着什么”等动机式、当前式或开放式标题。
```

### 4.3 自检增加逐词溯源

增加：

```text
空故事输出前，逐个检查标题中的时间、人物、事件、结果和变化：它们是否能在 payload 中直接找到？找不到就删除或改成开放式表达。
```

## 5. 第三轮验收标准

使用完全相同的空故事 payload 再测试。

必须满足：

- `source === "deepseek"`；
- 恰好 3 个选题；
- 恰好 1 个 `recommended: true`；
- 不出现任何 payload 未提供的时间、人物、场景、事件、评价、结果或变化；
- 每个标题都可以明确指出其 payload 依据；
- 不退化为泛知识题。

本轮至少要检查以下禁区：

- 不得出现“第一周/第一年”等未提供时间；
- 不得出现“一次/那天/一个瞬间”等未提供事件；
- 不得出现“学会/终于/后来”等未提供变化；
- 不得出现“被否定/被批评/搞砸”等未提供事实。
